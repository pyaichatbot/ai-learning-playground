# Basic vs Advanced Mode – UX & Product Boundary

## Basic Mode
**Purpose**
- Exploration
- Learning
- Zero pressure

**Characteristics**
- Multiple tools visible
- Flexible inputs
- No alerts
- No strong opinions
- Free forever

---

## Advanced Mode
**Purpose**
- System understanding
- Professional judgment
- Constraint awareness

**Characteristics**
- Single cockpit visible
- Opinionated UI
- Alerts instead of tips
- Explicit constraints
- Monetizable in future

---

## Boundary Rules
- No cockpit logic leaks into Basic Mode
- Advanced Mode is never required
- Advanced Mode explains, never fixes
- Basic Mode remains intact

---

## Upgrade Signaling (Future)
- Teaser copy only
- No hard paywalls initially
- “See how this behaves in production systems”
