## Documentation Workflow

### When Starting a New Story
1. Review the story in [USER-STORIES.md](./product/USER-STORIES.md)
2. Check [IMPLEMENTATION-TRACKER.md](./IMPLEMENTATION-TRACKER.md) for status
3. Follow the Story Loop process
4. Update tracker as you progress

### When Completing a Story
1. Mark story complete in [IMPLEMENTATION-TRACKER.md](./IMPLEMENTATION-TRACKER.md)
2. Add any learnings to [LEARNING-LOG.md](./LEARNING-LOG.md)
3. Update metrics and velocity
4. Document any technical debt

### When Encountering Issues
1. Document in [LEARNING-LOG.md](./LEARNING-LOG.md)
2. Update blocker status in [IMPLEMENTATION-TRACKER.md](./IMPLEMENTATION-TRACKER.md)
3. Add to best practices if applicable
