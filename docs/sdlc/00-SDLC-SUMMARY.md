## Development Process

### Story-Driven Development

We follow a **story-driven development process** with systematic testing and learning capture.

### Story Loop
1. **SELECT** - Review acceptance criteria
2. **IMPLEMENT** - TDD approach
3. **TEST** - Unit, integration, visual regression
4. **FIX EDGE CASES** - Boundary conditions
5. **LOG LEARNINGS** - Document insights
6. **CODE REVIEW** - Get approval
7. **MARK COMPLETE** - Update tracker

See [IMPLEMENTATION-TRACKER.md](../IMPLEMENTATION-TRACKER.md) for details.
